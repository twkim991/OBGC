# 러브레터 구현 규칙

## 기본 구성

- 2~6명이 플레이합니다.
- 최신판 21장 구성을 사용합니다: 첩자 2, 경비병 6, 성직자 2, 남작 2, 시녀 2,
  왕자 2, 재상 2, 왕 1, 백작부인 1, 공주 1.
- 라운드마다 카드 1장을 비공개로 제외합니다. 2인전에서는 추가로 3장을 공개 제외합니다.
- 자기 차례에 1장을 뽑고 손의 2장 중 1장을 사용합니다.

## 카드 효과

- 첩자(0): 라운드에서 첩자를 사용하거나 버리고 살아남은 플레이어가 단 한 명이면 호감 토큰 1개를 추가로 받습니다.
- 경비병(1): 경비병 이외의 인물을 추측해 맞히면 대상이 탈락합니다.
- 성직자(2): 대상의 손패를 행동한 플레이어에게만 한 번 보여줍니다.
- 남작(3): 서로의 손패를 비공개로 확인하고 값이 낮은 쪽이 탈락합니다.
- 시녀(4): 다음 자기 차례 시작 전까지 다른 플레이어의 효과를 받지 않습니다.
- 왕자(5): 자신을 포함한 대상이 손패를 버리고 새 카드를 뽑습니다. 공주를 버리면 탈락합니다.
- 재상(6): 덱에서 최대 2장을 더 확인하고 한 장만 남긴 뒤 나머지 순서를 정해 덱 아래에 둡니다.
- 왕(7): 대상과 손패를 교환합니다.
- 백작부인(8): 왕 또는 왕자와 함께 들고 있으면 반드시 백작부인을 사용합니다.
- 공주(9): 사용하거나 버리면 즉시 탈락합니다.

다른 모든 생존자가 시녀의 보호를 받으면 대상 카드 효과는 적용되지 않습니다. 왕자는 이 경우에도
자신을 선택할 수 있습니다.

## 라운드와 게임 승리

- 한 명만 남거나 덱이 비면 라운드가 끝납니다.
- 덱이 빈 경우 손패 값이 가장 높은 모든 플레이어가 라운드 호감 토큰을 받습니다.
- 목표 토큰은 2명 6개, 3명 5개, 4명 4개, 5~6명 3개입니다.
- 같은 라운드에 여러 플레이어가 목표에 도달할 수 있으며 공동 승리를 허용합니다.

## 공개 정보와 추리 UI

- 서버 공개 상태에는 개인 손패와 재상 선택 카드가 포함되지 않습니다.
- 성직자와 남작의 확인 결과는 해당 플레이어에게만 일회성 메시지로 전달합니다.
- 중앙에는 현재 행동에서 공개된 사용 카드와 효과로 버린 카드만 표시합니다.
- 누적 사용 카드 목록, 플레이어별 버린 카드 더미, 카드 수 카운터, 공개 행동 기록 패널은 구현하지 않습니다.
- 다음 행동이 공개되면 이전 행동 카드는 화면에서 교체됩니다. 플레이어는 직접 관찰하고 기억해 추리합니다.
- 2인전의 공개 제외 3장은 규칙상 현재 테이블에 공개된 정보이므로 라운드 동안 표시합니다.
